Angular routing

Its core part of angular. It enables to build SPA with multiple views and allow nagivation between these views.

You need to have base href="/" in head on index.html file which assumes that the app folder is the application root and that uses /.

app-routing.module.ts is the file where we provide routing for the application. It needs router module to perform the routing/navigating between the pages. The routes is of type Routes where all the routes are defined and this is set in NgModule where the Routes are set which means imported and exported where this can be imported in app.module.

Routes is a colletion of json objects, where json has two properties path(defines URL path for the route) and component which is the corresponding component for the particular route.

These routes cans be accessd or redirected from html using routerLink attribure.
Example : {{Example1}}

{{Example2}} : This element updates the angular to update the application view with the component for the selected route.
NOTE : Order of routes is important because router uses first match wins strategy.

Passing data between routes

If you wanna pass data from one component to another you use activated router interface.

Wild card route is a route when no route path matches its the last place where its gonna hit.
Syntex : path:"**" and define component. In our application we are redirecting to root component.

Child routes

Nested routes or child routes where we wanna expand the routes to parent routes. Here parent route is /angular-routing suppose we wanna add child route to this then it becomes :
a. /angular-routing/do_something_here_1
b. /angular-routing/do_something_here_2
We can also pass data to child routes which uses same way as activated routed as mentioned above.

Here if you see the url path its updated with child route.

Guards
Use route guard from restricting users from navigating to parts of the application without authorization. The following route guards are available in angular :
  1. CanActivate
  2. CanActivateChild
  3. canDeactivate
  4. Resolved
  5. CanLoad
A service guard is created which performs the authorization before uses is redirected to these views.
